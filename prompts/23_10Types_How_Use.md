## 🎯 페르소나 기반 시나리오 재구성

**페르소나 (Persona)**
* **직무:** 모바일 청첩장 제작 전문 업체 '디어모먼트'의 주니어 PM
* **성향:** 패션 에디터 출신으로 **비주얼과 타이포그래피, 사용자 감성(Vibe)**에 매우 예민함. "여기서 글자가 우아하게 떠올랐으면 좋겠어요"라고 말하지만, 개발자가 "그게 CSS 애니메이션인가요, 스크롤 트리거인가요?"라고 물으면 말문이 막힘.

#### **1. 배경 (Background)**
당신은 기존의 획일화된 템플릿에서 벗어나, 마치 **패션 화보를 보는 듯한 프리미엄 모바일 청첩장 서비스 '더 매거진(The Magazine)'**의 MVP를 기획 중입니다.
일주일 뒤, 개발팀 및 디자인팀과 함께 **'신규 테마 런칭 및 기능 개발 킥오프 미팅'**이 예정되어 있습니다.

현재 당신의 기획안에는 다음과 같은 **감각적이고 추상적인 아이디어**만 가득합니다.
* "메인 사진이 스크롤할 때 잡지 표지처럼 느낌 있게 줌아웃됐으면 좋겠다."
* "신랑신부 이름이 촌스럽지 않게, 등장할 때 페이드인 효과가 고급스러워야 한다."
* "하객들이 축하 메시지를 남길 때, UI가 복잡하지 않고 심플했으면 한다."

#### **2. 핵심 문제 (Problem)**
개발 리더로부터 기획서를 검토받은 후, 다음과 같은 **냉정한 피드백**을 받았습니다.

> **"PM님, '느낌 있게'나 '고급스럽게'는 코드로 짤 수가 없습니다.**
> 사진 용량 제한은 몇 MB로 할 건지, 스크롤 인터랙션은 어떤 라이브러리를 쓸 건지, 축하 메시지는 비회원도 작성 가능한지(DB 구조 문제)가 하나도 안 정해져 있어요.
> **이 상태로는 프론트엔드 인터랙션도, 백엔드 방명록 로직도 설계 못 합니다."**

구체적인 기능 명세와 데이터 로직이 정의되지 않으면, 디자인 시안만 예쁘게 나오고 **실제 기능은 작동하지 않는 '껍데기'만 만들어질 위기**입니다.

#### **3. 해결 과제 (Task)**
당신의 머릿속에 있는 화려한 디자인 아이디어를 개발자와 디자이너가 즉시 작업할 수 있는 **'모바일 청첩장 MVP 상세 요구사항 정의서(PRD)'**로 변환해야 합니다.

다음 내용들을 '개발 언어'에 가깝게 구체화해야 합니다.
1.  **기능적 요구사항:** 
사진 갤러리 업로드 방식(다중 선택, 크롭 기능), 방명록 CRUD(작성, 수정, 삭제) 로직, RSVP(참석 여부) 데이터 수집 방식 등.
2.  **비기능적 요구사항:** 
고화질 웨딩 사진 로딩 속도 최적화, 모바일 기기별 호환성(반응형), 개인정보(계좌번호) 보안 등.
3.  **정보 구조(IA):** 
인트로(메인) -> 초대글 -> 갤러리 -> 오시는 길 -> 방명록 -> 송금하기로 이어지는 구체적인 뎁스(Depth) 구조.
4. **클라이언트(청첩작 제작 의뢰자)의 요구사항을 반영**
- 웨딩 필름(영상, 이미지)을 추가해주세요.
- 웨딩 장소를 링크를 바로 지도 어플에 연동될 수 있도록 (카카오,네이버)
